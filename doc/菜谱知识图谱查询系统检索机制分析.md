# 菜谱知识图谱查询系统检索机制分析

## 一、数据存储结构

```python
# 加载的是 pickle 序列化的 NetworkX 图对象
with open(self.kg_path, 'rb') as f:
    graph = pickle.load(f)  # 这是一个 networkx.Graph 对象
```

**NetworkX 图的结构：**
- **节点 (nodes)**：每个节点有属性字典，如 `{'label': 'Dish', 'name': '小炒黄牛肉', 'prep_process': '...'}`
- **边 (edges)**：每条边有属性字典，如 `{'relation': 'USES_TECHNIQUE', 'amount': '适量'}`

---

## 二、检索方式详解

### 1. 正向属性查询（如："小炒黄牛肉的备菜过程"）

**流程：**
```
用户输入 → 解析出菜名 → 在字典中找节点ID → 读取节点属性
```

**核心代码：**
```python
# 第1步：建立菜名→节点ID的索引字典（在 _build_node_indices 中）
self.dish_nodes = {'小炒黄牛肉': 'node_123', '红烧肉': 'node_456', ...}

# 第2步：查找节点
dish_id = self.dish_nodes['小炒黄牛肉']  # 直接字典查找，O(1)

# 第3步：读取属性
node_data = self.graph.nodes[dish_id]    # 访问节点属性字典
attr_value = node_data['prep_process']   # 字典取值
```

**本质：Python 字典查找**

---

### 2. 正向关系查询（如："小炒黄牛肉用了哪些技法"）

**流程：**
```
用户输入 → 解析出菜名和关系类型 → 遍历该节点的所有出边 → 筛选匹配的关系
```

**核心代码：**
```python
# 遍历 dish_id 的所有出边
for _, target_id, edge_data in self.graph.edges(dish_id, data=True):
    edge_rel = edge_data.get('relation')  # 读取边的关系类型
    if edge_rel == 'USES_TECHNIQUE':      # 字符串匹配筛选
        target_node = self.graph.nodes[target_id]
        target_name = target_node.get('name')  # 获取关联实体名称
```

**本质：Python 遍历 + 字符串匹配**

---

### 3. 反向查询（如："哪些菜用了炝炒技法"）

**流程：**
```
用户输入 → 解析出关系类型和值 → 找目标实体节点 → 遍历所有指向该节点的入边
```

**核心代码：**
```python
# 第1步：找"炝炒"这个技法节点的ID
target_id = self.all_nodes_by_label['Technique']['炝炒']

# 第2步：遍历所有指向 target_id 的入边（反向查找）
for source_id, _, edge_data in self.graph.in_edges(target_id, data=True):
    edge_rel = edge_data.get('relation')
    if edge_rel == 'USES_TECHNIQUE':      # 筛选关系类型
        source_node = self.graph.nodes[source_id]
        dish_name = source_node.get('name')  # 得到菜名
```

**本质：Python 遍历入边 + 字符串匹配**

---

### 4. 模糊匹配（当精确查找失败时）

**核心代码：**
```python
from difflib import SequenceMatcher

def fuzzy_match(self, query, candidates):
    # 方法1：包含匹配（子串匹配）
    if query in candidate:  # 如"牛肉"在"黄牛肉"中
        score = 0.9

    # 方法2：SequenceMatcher 计算相似度
    score = SequenceMatcher(None, query, candidate).ratio()
    # 返回相似度最高的候选
```

**本质：字符串相似度算法**

## 三、与 Cypher 的对比

| 功能 | 当前脚本实现 | Cypher 实现 |
|------|-----------|------------|
| **属性查询** | `graph.nodes[id]['属性名']` | `MATCH (n) RETURN n.属性名` |
| **关系查询** | `for ... in graph.edges()` 遍历 | `MATCH ()-[r]->() RETURN ...` |
| **反向查询** | `for ... in graph.in_edges()` 遍历 | `MATCH ()<-[r]-() RETURN ...` |
| **模糊匹配** | `SequenceMatcher` 字符串算法 | `WHERE n.name CONTAINS 'xxx'` |
| **索引** | 手动构建 Python 字典 | 自动索引 + 查询优化器 |
| **查询语言** | 无，纯 Python 逻辑 | 声明式 Cypher 语言 |

---

## 四、总结

这个脚本的检索本质是：

> **Python 字典查找 + 列表遍历 + 字符串匹配**

具体来说：

1. **索引层**：手动构建 `dict` 做快速查找（菜名→ID、标签→节点集合）
2. **查询层**：用 `networkx.Graph.nodes[]` 和 `networkx.Graph.edges()` 访问图数据
3. **匹配层**：用 Python 字符串操作（`in`、`==`）和 `difflib.SequenceMatcher` 做匹配
4. **无查询语言**：没有 Cypher、没有 SQL、没有 SPARQL，全是 Python 代码硬编码逻辑

这种方式的优点是**轻量、无需数据库服务器、部署简单**；缺点是**查询逻辑复杂时代码臃肿、没有查询优化器、难以表达复杂图模式**。
